# 스터디 운영 규칙

---

## 스터디 방식

- 모든 멤버가 돌아가며 발표
- 사전에 합의된 스터디 날짜 **하루 전 23시 59분**까지 공부한 내용 업로드
  - 예) 스터디 날짜가 3월 1일이라면 → 2월 28일 23:59까지 업로드
- 기한 내 업로드하지 않을 경우 벌칙 적용
- (선택 사항) 다른 멤버가 올린 PR을 스터디 전에 한번씩 보고 리뷰 남기기!

---

## 날짜 선정 방식

- 모든 멤버의 합의하에 날짜 선정
- 선정된 날짜에 참여하지 못하더라도 스터디는 예정대로 진행

---

## 진행 방식

| 구분 | 방법           |
|------|--------------|
| 대면 | 스터디룸         |
| 비대면 | 구글미트 또는 디스코드 |

---

## GitHub 운영 방식

- Organization 레포지토리에 **개인 브랜치** 생성
- 주차별 디렉토리를 만들어 공부한 내용 업로드
- 업로드 후 **PR**을 통해 제출

```
브랜치명: {이름}/week{NN}
디렉토리: week{NN}/
예) 김동균/week01
```

---

## 벌칙

- 기한 내 미업로드 시 벌칙 적용 (커피빵)
