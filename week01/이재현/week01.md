# 1. 동시성과 병렬성의 차이
동시성은 "여러 작업을 번갈아 처리하는 구조"이고, 병렬성은 "여러 작업을 물리적으로 동시에 처리"하는 것

- 동시성 예시: 싱글코어 CPU에서 스레드 A, B를 빠르게 번갈아 실행하는 것 
- 병렬성 예시: 멀티코어 CPU에서 스레드 A, B를 각 코어에서 동시에 실행하는 것 

# 2. 멀티스레딩과 멀티프로세싱의 차이
Java는 멀티스레딩 기반으로 동작함. (Java 애플리케이션은 보통 하나의 JVM 프로세스에서 여러 스레드로 동작한다.)
멀티스레딩은 하나의 프로세스 내에서 여러 스레드가 작업을 수행하는 것, 
멀티프로세싱은 여러 프로세스가 작업을 수행하는 것이다. 

< 멀티스레딩 > <br>
프로세스 A <br>
├── 스레드 1  ─┐ <br>
├── 스레드 2  ─┼── Heap, Code 영역 공유 <br>
└── 스레드 3  ─┘   (Stack만 각자 소유) <br>

공유 메모리로 인한 동기화 문제가 발생할 수 있음.
하나의 스레드가 죽으면 프로세스 전체에 영향을 줄 수 있음.

< 멀티프로세싱 >
프로세스 A  (독립 메모리)
프로세스 B  (독립 메모리)
프로세스 C  (독립 메모리)

메모리는 독립적이라서 메모리 수준의 동기화 문제는 없음 
다만 멀티 프레세싱을 하는 경우 프로세스 간 통신(IPC)이 필요하므로, 일반적으로 멀티스레딩보다 통신 비용이 크다. 
단, 외부 공유 자원(DB, 파일 등)을 사용하면 마찬가지로 동시성 이슈가 발생 가능

# 3. 자바에서 스레드와 cpu의 연관관계?

OS 스케줄러는 자바의 스레드를 CPU 코어에 배치한다.

코어가 1개인 경우에는 여러 스레드가 동시에 실행되지 않고, 아주 짧은 단위로 번갈아 실행된다.  
인간이 느끼기에 거의 동시의 수준인거지, 실제 동시에 실행되는 것은 아니라는 것이다. 
따라서 단일 코어 환경의 멀티스레딩은 동시성(concurrency)에 가깝다.

반면 코어가 여러 개인 경우에는 여러 스레드가 서로 다른 코어에서 실제로 동시에 실행될 수 있다.  
따라서 멀티코어 환경에서는 병렬성(parallelism)이 가능하다.

* 시분할 기반 : cpu 시간을 잘게 나눠서 여러 작업이 조금씩 번갈아 실행되게 하는 방식
즉, 스레드 자체는 같지만 실행 환경에 따라 단일 코어에서는 **시분할 기반**의 동시성으로, 멀티코어에서는 실제 병렬 실행으로 동작할 수 있다.
결국 CPU intensive한 작업은 사용할 수 있는 코어 수가 많을수록 유리하다.

Java 21의 virtual thread는 물리적인 코어 수를 늘리는 기술이 아니다.  
일반 스레드처럼 작성하되, OS 스레드를 훨씬 적게 사용하면서 많은 동시 작업을 처리할 수 있게 만든 가벼운 스레드이다.

즉, 자바의 virtual thread는 CPU 연산을 더 빠르게 만드는 기술이 아니라,
I/O 대기 시간 동안 OS 스레드를 효율적으로 재사용해 동시 처리량을 높이는 기술이다.

이런 점에서 I/O 대기 작업을 효율적으로 처리한다는 관점에서는 node.js의 libuv와 비슷한 면이 있다.
흔히 js가 싱글 스레드로 동작한다고 말하지만, 정확히는 js 코드를 실행하는 메인 이벤트 루프가 단일 스레드로 동작하는 것이다. 
내부적으로는 libuv와 운영체제의 비동기 I/O 메커니즘을 통해 I/O 작업을 효율적으로 처리한다.
다만 Node.js가 이벤트 루프 중심의 비동기 모델이라면, Java의 virtual thread는 동기식 코드 스타일을 유지하면서도 많은 작업을 가볍게 처리할 수 있게 해준다는 차이가 있다.
